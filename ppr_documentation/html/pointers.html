A few important points

Besides the subjects covered in the Getting Started page, this page contains a few important pieces of information about how PoProofRead works.

The PoProofRead files

After opening a file and saving the ongoing work for the first time, two new files has now been created, in the same folder that the original file was placed in. The files are named like this:

example.podiff

This is the file with the original content that needs proofreading. This file should only be opened when starting the work on this file's content.

example.podiff.ppr

The is PoProofRead's internal representation of the proofreading. It contains all the content of the original podiff, the comments and various information about the current state of the proofreading. This is the file that should be opened for resuming previous work on a proofreading.

example.podiff.ppr.out

This file contains the output of the proofreading. It contains the podiff parts that has been commented on and their comments. It is ready to send back to the author of the podiff as is.

Saving the current work

The current work will automatically be saved everytime you move away from a changed comment, when you manually save, or when you shut down the program.

In-line mode for a comment

In the in-line commenting mode, only the content of the commenting field will be written to the output. This means that the commenting field should contain the diff part that is being commented on as well as your comment. If you activate the in-line commenting mode while the commenting field is empty the diff part will be automatically copied to the commenting field. Otherwise you will have to copy it manually.
